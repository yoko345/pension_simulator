# Pull Request の書き方（このリポジトリの定型）

## 目的

PR 説明を毎回同じ構成にそろえ、将来の自分が履歴を読みやすくする（個人開発向けの最小構成）。

## 使い方

### GitHub 上で PR を作る場合

リポジトリに [`.github/pull_request_template.md`](../.github/pull_request_template.md) があるため、**Pull Request を新規作成すると本文にこのテンプレートが自動で入ります**。プレースホルダ（`<!-- ... -->`）を置き換え、チェックリストを更新してください。

### 手元やエディタだけで下書きする場合

同じ内容をコピーするなら、**テンプレートの原本**は次のファイルです。

- [`.github/pull_request_template.md`](../.github/pull_request_template.md)

## 必須セクション（見出し名は固定）

| 見出し | 書くこと |
|--------|----------|
| 概要 | 何のための PR か、結果として何が変わるか |
| 背景 | なぜ必要か、Issue や文脈 |
| 変更内容 | 具体的な差分の説明（パス・挙動） |
| 動作確認 | 実施した確認とチェックボックス |

## AI（Cursor）向けメモ

PR 本文やドラフトを依頼するときは、**上記の見出し構成に従う**こと。詳細ルールは `.cursor/rules/pull-request-format.mdc` を参照。
