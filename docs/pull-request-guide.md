# Pull Request の書き方（このリポジトリの定型）

## 目的

PR 説明を毎回同じ構成にそろえ、将来の自分が履歴を読みやすくする（個人開発向けの最小構成）。

GitHub の **PR テンプレート（`.github/pull_request_template.md`）は使わない**。本文は必要に応じて Cursor に依頼してコピペするか、下記見出しに沿って手で書く。

## 必須セクション（見出し名は固定）

| 見出し | 書くこと |
|--------|----------|
| 概要 | 何のための PR か、結果として何が変わるか |
| 背景 | なぜ必要か、Issue や文脈 |
| 変更内容 | 具体的な差分の説明（パス・挙動） |
| 動作確認 | 実施した確認とチェックボックス |

## AI（Cursor）向け

PR 本文のドラフトを依頼するときのルールは `.cursor/rules/pull-request-format.mdc` を参照。
