# 1. 概要

本設計は、年金受給者の手取り計算における
**介護保険料を区の制度に基づき段階制で再現するための実装仕様**である。

# 3. 第1号被保険者（65歳以上）

---

## 3.1 基本構造

* 段階制（最大20段階）
* 基準額：73,300円
* 所得・世帯状況により決定

---

## 3.2 実装用テーブル（簡略版）

```ts
const CARE_TABLE = [
  { level: 1, maxIncome: 809000, fee: 20900, condition: "世帯非課税" },
  { level: 2, maxIncome: 1200000, fee: 31600, condition: "世帯非課税" },
  { level: 3, maxIncome: Infinity, fee: 50300, condition: "世帯非課税" },

  { level: 4, maxIncome: 800000, fee: 62400, condition: "本人非課税・世帯課税" },
  { level: 5, maxIncome: 1200000, fee: 73300, condition: "基準額" },

  { level: 6, maxIncome: 1200000, fee: 84300 },
  { level: 7, maxIncome: 2100000, fee: 91700 },
  { level: 8, maxIncome: 3200000, fee: 102700 },
  { level: 9, maxIncome: 4000000, fee: 124700 },

  { level: 10, maxIncome: Infinity, fee: 132000 }
];
```

---

## 3.3 段階判定ロジック

```ts
function getCareLevel(params) {
  const income = params.income;
  const householdTaxExempt = params.householdTaxExempt;
  const selfTaxExempt = params.selfTaxExempt;

  // 非課税世帯
  if (householdTaxExempt) {
    if (income <= 809000) return 1;
    if (income <= 1200000) return 2;
    return 3;
  }

  // 世帯課税・本人非課税
  if (selfTaxExempt) {
    if (income <= 800000) return 4;
    return 5;
  }

  // 課税世帯
  if (income < 1200000) return 6;
  if (income < 2100000) return 7;
  if (income < 3200000) return 8;
  if (income < 4000000) return 9;

  return 10;
}
```

---

## 3.4 保険料計算

```ts
function calcCareInsurance65Plus(params) {
  const level = getCareLevel(params);
  const row = CARE_TABLE.find(r => r.level === level);
  return row ? row.fee : 0;
}
```

# 5. 全体関数

```ts
function calcCareInsurance(params) {
  const { age } = params;

  if (age < 40) return 0;

  if (age < 65) {
    return 0; // 国保に含める
  }

  return calcCareInsurance65Plus(params);
}
```

# 8. 注意書き（必須表示）

---

## ■ 年金収入のみを前提とした計算について

本シミュレーションでは、**年金収入のみ**を前提として介護保険料を算出しています。

実際には以下の要素により、介護保険料の段階が変動します：

* 給与収入
* 不動産収入
* 配偶者の所得
* 世帯全体の課税状況

これらがある場合、本シミュレーション結果と実際の保険料は異なる可能性があります。

---

## ■ 段階判定の簡略化について

本ツールでは区の制度を基にしていますが：

* 本来は20段階ある区分を一部簡略化
* 住民税判定を簡易ロジックで代替

しています。

そのため、実際の保険料と差異が生じる可能性があります。

---

## ■ 制度変更について

本モデルは2026年時点の制度を前提としています。

将来的に制度改定が行われた場合、結果は変動します。
